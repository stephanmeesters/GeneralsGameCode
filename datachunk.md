DataChunk overview
- `GeneralsMD/Code/GameEngine/Include/Common/DataChunk.h` and `Source/Common/System/DataChunk.cpp` implement a chunked binary serialization format. Each chunk stores `[id][version][size][data]`, using a table of contents (TOC) tagged `CkMp` to map chunk names to numeric ids.
- `DataChunkOutput` writes chunks to a temp file, fills in sizes on close, then prepends the TOC and copies the buffered data to the real output stream in its destructor. It provides typed writers for ints, reals, byte arrays, ASCII/Unicode strings, dicts, and name keys.
- `DataChunkInput` reads the TOC, then walks chunks invoking registered parsers scoped by parent chunk. It tracks remaining bytes per chunk so nested data can be skipped safely and exposes typed readers for the same primitives.
- Map load/save: `WorldHeightMap` registers parsers for `HeightMapData`, `BlendTileData`, `WorldInfo`, `ObjectsList`, `PolygonTriggers`, `SidesList`, `GlobalLighting`, etc., to build the in-memory world (`GeneralsMD/Code/GameEngineDevice/Source/W3DDevice/GameClient/WorldHeightMap.cpp`). `TerrainLogic::loadMap` reads waypoint links from `WaypointsList` (`GeneralsMD/Code/GameEngine/Source/GameLogic/Map/TerrainLogic.cpp`).
- Map metadata: `SidesList::WriteSidesDataChunk`/`ParseSidesDataChunk` serialize sides, teams, build lists, and per-side scripts inside a `SidesList` chunk (`GeneralsMD/Code/GameEngine/Source/GameLogic/Map/SidesList.cpp`).
- Triggers: `PolygonTrigger::WritePolygonTriggersDataChunk`/`ParsePolygonTriggersDataChunk` store polygon trigger definitions (`GeneralsMD/Code/GameEngine/Source/GameLogic/Map/PolygonTrigger.cpp`).
- Scripting: the script engine writes/reads `PlayerScriptsList`, `ScriptList`, `Script`, and nested condition/action chunks to persist mission/AI scripts (`GeneralsMD/Code/GameEngine/Source/GameLogic/ScriptEngine/Scripts.cpp`; declarations in `.../Include/GameLogic/Scripts.h`).
- Misc: map preview/metadata readers declare chunk parsers in `GeneralsMD/Code/GameEngine/Include/GameClient/MapUtil.h`.
